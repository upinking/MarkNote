# MarkNote

一个轻量的 macOS / Windows Markdown 笔记软件。

## 功能

- 打开 `.md` / `.markdown` 文件
- 保存当前 Markdown 到本地文件
- 默认进入所见即所得写作模式，也可以切回 Markdown 源码编辑
- 左侧根据 `#`、`##`、`###` 自动生成目录
- 点击目录跳转到对应标题
- 右侧实时预览 Markdown
- 更完整的 Markdown 预览：表格、引用、代码块、任务列表等
- 多主题、四种视图模式、自动换行、界面语言设置
- AI Agent 侧边栏：可总结、润色、整理结构、续写，并生成可预览后应用的修改草案
- 点击顶部文件名查看最近打开的文件并快速切换
- 自动保存草稿，异常退出后可恢复
- 右键文件名可另存为、导出 PDF、显示文件、重命名、删除文件
- 新建 / 打开 / 关闭前的未保存提醒
- 系统习惯快捷键：macOS 用 `Cmd`，Windows 用 `Ctrl`

## 使用说明

### 打开和保存

点击顶部的“打开”可以选择本地 `.md` 或 `.markdown` 文件。编辑后点击“保存”会写回原文件；如果是新建笔记，第一次保存会让你选择保存位置。

顶部当前文件名可以点击展开最近打开的文件列表。切换到其他文件前，如果当前内容还没保存，MarkNote 会先询问你保存、不保存或取消。

也可以右键顶部当前文件名，选择“重命名”来修改本地文件名。还没有保存到本地的新笔记，需要先保存后才能重命名。

右键顶部当前文件名还可以使用“另存为”“导出 PDF”“在文件夹中显示”和“删除文件”。删除会把文件移到系统废纸篓或回收站。

编辑时 MarkNote 会自动保存一份本地草稿。如果上次编辑没有正常保存，下次启动时会询问是否恢复草稿。

### 视图模式

右上角的视图按钮会在四种模式之间循环：

- 默认：逐行写作模式。当前正在编辑的行显示 Markdown 源码；按回车进入下一行后，上一行会自动显示成排版后的效果。
- 编辑：显示 Markdown 源码，适合精确修改语法。
- 预览：左侧目录、中间源码编辑区、右侧预览同时显示。
- 阅读：隐藏编辑区，只保留目录和预览，适合检查成稿。

### 目录

目录会自动读取 `#`、`##`、`###` 标题。点击目录项会跳转到对应标题；一级标题左侧的小三角可以展开或收起子标题。

### 自动换行

“自动换行”在设置里。它只影响编辑区的显示方式，不会改动 Markdown 文件内容。

### 自动转换待办框

设置里可以打开“自动转换待办框”。打开后，编辑时会把 `- 【】 任务`、`- [] 任务`、`- 【x】 任务` 自动转换成标准 Markdown 待办项，也就是 `- [ ] 任务` 或 `- [x] 任务`，方便文件在其他 Markdown 软件里继续识别。

### 主题和语言

点击右上角主题按钮可以轮换主题。更多设置可以在右上角“设置”里修改，包括界面语言、主题和自动换行。

### AI Agent

右上角的 “AI” 按钮会打开 AI 侧边栏。打开后，中间会自动切到“默认”写作模式，右侧显示对话框。

第一次使用前，需要在“设置”里填写 AI 配置：

- Provider：可以选择 OpenAI 或 DeepSeek
- Model：默认是 `gpt-4.1-mini` 或 `deepseek-chat`
- API Key：保存在本机浏览器存储里，不会写进 Markdown 文件

AI 可以直接回复，也可以生成一份完整 Markdown 草案。草案不会自动覆盖当前笔记，只有点击“应用修改”后，当前笔记才会被替换，并进入正常的未保存状态。

### 使用说明

设置菜单底部的“使用说明”会打开这个 `README.md`。关闭说明后，软件会自动回到你之前正在编辑的笔记。

## 快捷键

- `Cmd/Ctrl + N`：新建
- `Cmd/Ctrl + O`：打开
- `Cmd/Ctrl + S`：保存
- `Cmd/Ctrl + Shift + S`：另存为
- `Cmd/Ctrl + E`：导出 PDF
- `Cmd/Ctrl + P`：轮换视图模式（默认 / 编辑 / 预览 / 阅读）
- `Cmd/Ctrl + D`：轮换主题
- `Cmd/Ctrl + L`：切换自动换行

## 运行

```bash
npm install
npm run dev
```

`npm run dev` 会先准备编辑器脚本，再启动 Electron。

## 打包

当前机器是什么系统，就会优先打出对应系统的安装包：

```bash
npm run build
```

在 macOS 上打 macOS 包，在 Windows 上打 Windows 包。生成文件会在 `dist/` 目录里。

如果在 macOS 上遇到 electron-builder 的签名限制，可以用这个轻量打包命令直接生成 `.app`：

```bash
npm run package:mac
```

## 文件结构

- `app/`：界面、编辑器逻辑、样式
- `electron/`：桌面窗口、打开文件、保存文件
- `package.json`：运行和打包配置

## 小提示

`app/renderer.js` 是主要逻辑文件。你可以把它理解成“编辑器的大脑”：它负责读取输入、生成目录、渲染预览、处理按钮点击。
